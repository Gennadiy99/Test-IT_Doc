Урок по Git end Git-Hub

Добавление тега для нового commit

Для первого входа в сохранение данных на Git end Git-Hub нужно внести команды:

  1. git init
  2. git add .
  3. git commit -m "first commit" .
  4. git branch -M main
  5. git remote add origin "ссылка на репозиторий Git-Hub"
  6. git push -u origin main

Для последующих сохранений нужно только команы:

  1. git add .
  2. git commit -m "new commit"
  3. git push

Удаление репозитория Git-Hub выполняется непосредственно в самом репозитории выбрав "Seting" настройки. В самом конце списка есть раздел для удаления репозиторя.

Так же нажав клавишу "code" можно выбрать опцию ZIP архив и скачать код на компьютер

Для копирования репозитория из Git-Hub можно воспользоваться кнопкой копирования ссылки. После чего нужно на компьютере открыть терминал "wind"+r и внести команду git clone (и вставить скопированную ссылку). После этого создастся папка c файлом html + git, на раб.столе или диске "C" название папки репозитория с Git-Hub

В можно создать новый терминал возле + стрелка и выбрать Git Bash (терминал с подсветкой)

Как и другие терминалы (этот терминал с цветной расцветкой) в нем можно прописывать команды git add и можно узнавать его статус git status. Данная команда покажет если какието изменения в репозитории

Далее нужно отправить эти изменения в Git-Hub или "КНОПКОЙ" в граф.итерф или через меню "push" (отправить) или в командной строке git push

Есть команда "git pul" или в меню "Pul" вытягивание. При этой команде мы получаем изменения внесенные другими разработчиками из Git-Hub

Алиасы это сокращенные пользовательские команды позволяющие использовать стандартные команды Git

git status --short /выводит сокращенную информацию по Статусу документа/

В терминале мы можем прописать команду "git status --short" в место команды которая выведет полное описание втатуса "git status"

Если произошли какието не зафиксированные изменения то в Статусе они будут подсвечиваться красной буквой "М"

Если мы в Граф.ред. нажмем на "+" это то же самое что в терминале команда "git add ." Тоесть изменения будут добавленны в "стэйтж" - После чего при команде "git status --short" буква "М" будет подсвечиваться зеленой.

-- Далее нужно комитить и пушить что бы изменения попадали в репозиторий Git-Hub

Команда git log --oneline --graph --decorate --all - выдаст отобранную инормацию по камитам(без лишних данных)

Есть список команд для терминала, которые дают нам более точную информацию, такую как выаодит Git-Graph

git branch -M main - Может переименовывать ветки, НО может и СОЗДАВАТЬ.(расмотрим это позже.)

Мы можем открыть файл Git config которая находится на ПК в User, сделаем это через VS code. В этом файле можно создавать "alias - Алиас" - сокращенные команды. 

      [alias]
      s = status --short
      l = log --oneline --graph --decorate --all
      g = log --graph --abbrev-commit --decorate --all --format=format:'%C(bold blue)%h%C(reset) - %C(bold cyan)  %aD%C(dim white) - %an%C(reset) %C(bold green)(%ar)%C(reset)%C(bold yellow)%d%C(reset)%n %C(white)%s%C  (reset)'
      br = branch
      co = checkout

После этого мы можем в терминали писать сокращенную команду "git s" - это равнозначно "git status"

Можно сохранять сокращенную команду в таком уточнени как вам удобно, допустим: 

        br = branch -M main 
        Теперь команда br будет переименовывать ветку в "main"

4.10 Отмена изменений

Если просто сохранили изменения в файле (не отправили в "Промежуточные" через "add .")

Сохраненное изменение в Файле html можно отменить в Граф.инт. "Стрелочкой" на кнопке "Изменения"

Так же можно отменить через Меню 3 точки- Изменения- Отменить все изменения.

Так же можно отменить изменения в ком.стр. написав:

  • "git checkout -- ." - отм.изм. во всех файлах
  • "git checkout -- index.html" - отм.изм. в файле index.html
Важно учитывать что есть пробелы,перед дефис и после.

Таким способом мы отменяем сохранения в файле (html/css ...)

Если мы сохранили изменение и добавили их в "промежуточные изменения".

Выполнили это сохранение через:

  • Через командную строку "git add ."
  • Через граф.инт "+" - "Индексировать изменения"
Отменить можем:
  • в Граф.интерф. нажатием на " - " минус. На надписи "Промежуточное значение"
  • Меню 3 точки - "Изменения" - "Отменить Промежуточные изменения"
  • в командной строке "git reset ." или "git reset inde.xhtml" (если изменения были только в нем)

Если сохранили изменения в "commit"

Сохранение в commit делается:

  • терминал командой git commit -m "new commit"
  • Написания названия комита в поле "Сообщение" + кнопка "Фиксация"в Граф.интр.

Отмена делается commita делается:

  • ком.строке git reset -- hard HEAD^1 (жесткий сброс 1 комит)
